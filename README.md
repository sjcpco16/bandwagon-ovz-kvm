## 搬瓦工 OVZ 切换 KVM 这件事，先说结论

## OVZ 和 KVM 到底差在哪

在讲怎么迁之前，先搞清楚为什么要换。

OpenVZ 的架构特点是共用宿主机内核，资源隔离比较弱，有一些东西跑不了：BBR 加速装不上，锐速也不行，自定义内核更不用想，Windows 系统也装不了。用来跑基础应用问题不大，但想折腾或者要用高级功能的话，卡点很多。

KVM 是完整虚拟化，每台机器有独立的内核和完整的硬件模拟层。带来的好处是：

- 可以装 BBR 或者后台直接选带 BBR 的系统镜像
- 支持自定义 ISO，包括 Windows
- 支持通过 KiwiVM 后台一键迁移到 CN2、CN2 GIA 等高端机房
- 内核级别的控制权更完整

价格上，两者在搬瓦工历史上是一个价位，OVZ 没有便宜多少。所以当初就该选 KVM，现在 OVZ 也全退了，这个问题算是被市场自然解决了。

## 实际迁移步骤：OVZ 数据搬到 KVM

因为每个人用的服务不一样，这里列三类常见场景，按需对号入座。

### 准备工作

1. 先买好新的 KVM 套餐，拿到 IP 和 SSH 密码
2. 同时开两个 SSH 窗口——一个连 OVZ 旧机，一个连 KVM 新机
3. 在两台机器上都安装 rsync

bash
# Ubuntu / Debian
apt update && apt install -y rsync

# CentOS / Rocky Linux
yum install -y rsync


### 场景 A：纯文件迁移

在 OVZ 机器上，把要迁的目录打包：

bash
cd /home
tar -czvf backup.tar.gz example/


然后用 rsync 把压缩包传到 KVM 新机（在 OVZ 上执行，替换 IP 和路径）：

bash
rsync -avz --progress backup.tar.gz root@新机IP:/home/


到 KVM 新机上解压：

bash
cd /home
tar -xzvf backup.tar.gz


搞定。rsync 支持断点续传，大文件也不怕中途断开。

### 场景 B：网站迁移（Nginx + MySQL）

**第一步：导出数据库**

在 OVZ 上执行：

bash
mysqldump -u root -p 数据库名 > site_db.sql


**第二步：传文件和数据库**

把网站文件和 sql 文件一起传到新机：

bash
rsync -avz /var/www/你的网站目录/ root@新机IP:/var/www/你的网站目录/
rsync -avz site_db.sql root@新机IP:/root/


**第三步：在 KVM 新机上导入数据库**

bash
mysql -u root -p 数据库名 < /root/site_db.sql


**第四步：调整配置**

在新机上，检查 Nginx 配置里的路径是否正确，更新数据库连接配置（`wp-config.php` 或其他配置文件）里的 IP / 密码，重启服务：

bash
systemctl restart nginx
systemctl restart mysql


### 场景 C：跑了其他应用（Docker 等）

如果是 Docker 部署的应用，把 `docker-compose.yml` 和挂载的数据卷目录传过去就好，然后在新机上重新 `docker compose up -d`。

## IP 会变吗？怎么处理

会变。新买的 KVM 套餐是新的 IP，OVZ 的老 IP 无法迁移过来。

如果有域名绑定了 OVZ 的 IP，在 KVM 上跑起来之后更新一下域名解析就行，DNS 传播一般几分钟到几小时。

如果你的应用没有绑定域名，那 IP 变化会直接影响访问地址，提前通知使用方或者做好重定向。

## 常见问题

**Q：搬瓦工 OVZ 套餐还能用吗？**

已经在跑的 OVZ 机器可以用到期，但续费不了，到期即停。搬瓦工官方早已全面停售 OVZ 套餐。

**Q：OVZ 迁移到 KVM，原来的 IP 能保留吗？**

不能。两台机器是独立的，KVM 新机会分配新的 IP。

**Q：KVM 入门套餐和 CN2 GIA-E 差在哪？**

线路差别最大。KVM 入门套餐走的是普通国际线路（默认机房），切到 CN2 机房后流量变为三分之一，而且带宽相对窄。CN2 GIA-E 直接走 CN2 GIA 高端线路，带宽 2.5 Gbps，晚高峰抗拥堵能力更强。预算够的话，CN2 GIA-E 用起来差别是能感受到的。

**Q：迁移过程中业务会中断吗？**

用双机并行迁移的方式可以把中断时间控制到最短。OVZ 和 KVM 同时跑，数据同步完成、测试没问题之后，再切换域名解析，中断只在 DNS 切换那一段，通常几分钟。
